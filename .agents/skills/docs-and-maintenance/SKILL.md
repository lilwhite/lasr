# docs-and-maintenance

## Purpose

Mantener la documentación operativa alineada con el estado real del repositorio. Garantizar que las instrucciones de despliegue, uso y troubleshooting sean precisas y accionables.

## When to Use

- Actualización de documentación
- Cambios en flujos de despliegue
- Nuevas features operativas
- Revisión de coherencia documental

## Scope

- `docs/DEPLOY.md`
- `README.md`
- `AGENTS.md`

## Input / Target Files

- `docs/DEPLOY.md` - Instrucciones de despliegue
- `README.md` - Página principal del repo
- `AGENTS.md` - Instrucciones para agentes

## Rules

1. Explicar estrategia actual de despliegue
2. Indicar cómo activar GitHub Pages con Actions
3. Indicar cómo revisar fallos
4. Mantener documentación breve, clara y accionable
5. Actualizar coherencia si cambia el flujo
6. Incluir troubleshooting básico
7. Explicar cómo actualizar contenido
8. Documentar cambios relevantes

## Checklist

- [ ] DEPLOY.md explica GitHub Actions
- [ ] README.md coherente con estructura real
- [ ] AGENTS.md actualizado con skills
- [ ] Instrucciones de activación claras
- [ ] Troubleshooting básico incluido
- [ ] Pasos de publicación documentados
- [ ] Coherencia entre documentos

## Expected Output

- Documentación actualizada y coherente
- Instrucciones claras y accionables
- Coherencia entre archivos documentales

## What NOT to Do

- No escribir documentación excesiva
- No inventar procedimientos que no existen
- No contradecir el estado real del repo
- No omitir pasos críticos de despliegue
- No crear documentación para features que no existen
- No duplicar información innecesariamente
